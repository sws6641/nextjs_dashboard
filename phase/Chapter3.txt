#2. Chapter3. 폰트, 이미지 최적화
- #2-1. next/font를 활용한 커스텀 폰트 추가
    외부 파일을 통한 커스텀 폰트(폰트 파일을 직접 다운받아서 import 해서 사용하는 방식)
    html 파일을 통해 작성하는 방식
    → 폰트 파일을 가져오고 로딩하는 과정에서 애플리케이션 성능에 영향을 미칠 수 있음

    next.js는 next/font라는 모듈을 통해서 애플리케이션을 빌드(build)할 때 폰트 파일을 미리 다운로드를 해둠
    → 미리 다운되었기 때문에 사용자가 페이지에 접근할 때 폰트 때문에 추가적인 네트워크 요청을 수행할 필요가 없어짐

- #2-2. next/image를 활용한 이미지 추가

<img /> 태그를 사용하였을 경우 발생할 수 있는 문제
1. 스크린 사이즈(화면 사이즈)에 따른 이미지 반응형 처리
2. 다른 기기간(모바일, 태블릿, PC)이미지 사이즈를 별도로 적용
3. 이미지가 로딩될 때 이미지 사이즈에 따라 화면이 밀리는 layout shift 현상 방지 처리

<Image />를 사용할 경우에는 위와 같은 처리가 자동으로 최적화됨
